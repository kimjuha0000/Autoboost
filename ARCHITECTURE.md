# Autoboost 아키텍처 설명서

이 문서는 "Autoboost" 프로젝트가 왜 **React(Vite)를 사용한 프론트엔드**와 **Express.js 기반의 범용 백엔드(서버리스)** 구조로 설계되었는지 그 기술적 배경을 설명합니다.

---

### 1. 왜 프론트엔드와 백엔드를 분리했나요?

이전의 Next.js 프로젝트에서 빌드 오류가 발생했던 경험을 바탕으로, 더 안정적이고 표준적인 구조를 채택하기 위해 프론트엔드와 백엔드의 역할을 명확히 분리했습니다.

-   **프론트엔드 (Frontend)**: 사용자의 눈에 직접 보이는 모든 화면(UI)을 담당합니다. 이번 프로젝트에서는 React(Vite)를 사용하여 '사장님 대시보드' 화면을 만듭니다. 사용자와의 상호작용에만 집중하므로 개발 및 유지보수가 용이합니다.

-   **백엔드 (Backend)**: 데이터 저장, 처리, 보안 등 눈에 보이지 않는 서버 로직을 담당합니다. QR 코드 스캔 기록을 저장하고, 대시보드가 요청하는 데이터를 제공하는 등의 역할을 수행합니다.

이러한 분리 구조는 현대 웹 서비스에서 가장 널리 쓰이는 **"Jamstack 아키텍처"**의 기본 형태로, 각자의 역할에만 집중하여 **안정성과 확장성**을 크게 높일 수 있습니다.

### 2. 왜 백엔드로 "서버리스(Serverless)"를 선택했으며, 왜 범용 Express.js인가요?

전통적인 백엔드는 항상 켜져 있는 서버 컴퓨터(예: AWS EC2)를 필요로 하지만, 이는 비용과 관리 부담이 큽니다. "서버리스" 방식은 이러한 단점을 극복하기 위해 등장했습니다.

-   **서버리스(Serverless)란?**: 서버가 없다는 뜻이 아니라, 개발자가 서버를 직접 관리할 필요가 없다는 의미입니다. 클라우드 제공 업체(이번엔 Netlify/Vercel)가 알아서 서버를 관리하고, **오직 API가 호출될 때만 순간적으로 코드를 실행**시킨 후 다시 잠재웁니다.

-   **Netlify Functions 및 Vercel Serverless Functions**: Netlify와 Vercel 모두 서버리스 기능을 제공합니다. 우리는 Node.js 기반의 Express.js 애플리케이션을 백엔드로 구축하여, 이 Express 앱을 각 플랫폼의 서버리스 함수 형태로 실행할 것입니다. 이는 두 플랫폼 모두에 유연하게 배포할 수 있는 가장 범용적인 방법입니다.

#### 서버리스의 장점:
1.  **비용 효율성**: API가 호출된 시간만큼만 아주 적은 비용이 부과됩니다. 트래픽이 거의 없는 초기 단계에서는 사실상 무료로 운영할 수 있습니다.
2.  **관리 부담 없음**: 서버가 다운될 걱정, 보안 업데이트, 트래픽에 따른 서버 증설(스케일링) 등을 모두 플랫폼(Netlify/Vercel)이 자동으로 처리해줍니다. 우리는 오직 코드 개발에만 집중하면 됩니다.
3.  **통합된 개발 환경 및 유연성**: 프론트엔드와 백엔드 코드를 하나의 프로젝트에서 관리하고, Netlify 또는 Vercel을 통해 한 번에 배포할 수 있어 개발 및 운영이 매우 편리합니다. 특히, Express.js 기반이므로 필요시 다른 Node.js 호스팅 서비스로 쉽게 이전할 수도 있습니다.

### 3. 전체 시스템 흐름

이러한 아키텍처를 바탕으로, 우리 프로젝트는 다음과 같이 동작합니다.

```
┌───────────────┐   (1) QR 스캔   ┌──────────────────────┐   (2) 데이터 기록 요청   ┌────────────────┐
│   사용자 (고객) │───────────────>│ Express.js API         │────────────────────>│ data/events.json │
└───────────────┘                 │ (Backend, Serverless)│<────────────────────┤ (데이터베이스 역할)│
                                  └──────────────────────┘   (4) 데이터 조회     └────────────────┘
                                            ^
                                            │ (3) 데이터 요청
                                            │
                                  ┌──────────────────────┐
                                  │   React 대시보드 App   │
                                  │   (Frontend on Netlify/Vercel)│
                                  └──────────────────────┘
```

1.  **QR 스캔**: 고객이 QR 코드를 스캔하면 Express.js 기반의 백엔드 API가 호출됩니다.
2.  **데이터 기록**: 백엔드 API는 스캔 시간, IP 등의 정보를 `data/events.json` 파일에 기록합니다.
3.  **데이터 요청**: 사장님이 대시보드 페이지에 접속하면, React 앱이 백엔드 API에 스캔 기록 데이터를 요청합니다.
4.  **데이터 조회 및 표시**: 백엔드 API는 `data/events.json` 파일에서 데이터를 읽어 React 앱에 전달하고, 대시보드는 이 데이터를 표 형태로 красиво하게 보여줍니다.
